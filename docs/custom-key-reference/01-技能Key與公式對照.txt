技能 Blackboard Key 與公式對照
==============================

一、目前通用公式直接識別的標準 key
----------------------------------

CHANGE_attackType
  意義：覆蓋主傷害流的傷害類型。
  custom 常用值：物理、法術、真實、治療、不攻擊。
  公式：物理套用防禦；法術套用法抗；治療與不攻擊的技能傷害為 0。
  真實傷害：不受防禦與法抗影響，但仍套用技能與天賦的最終傷害倍率。
  注意：這通常是 custom 輸出 key，不是可靠的原始 JSON 通用 key。

atk
  意義：技能期間攻擊力加成比例。
  單位：小數比例，0.5 = 攻擊力 +50%。
  公式：基礎攻擊力 * (1 + 技能 atk + 特殊天賦 attack + 模組／分支 atk)。
  注意：這不是 atk_scale，也不是直接增加固定攻擊力。

atk_scale
  意義：攻擊倍率／單次攻擊縮放。
  單位：倍率，1.5 = 造成相當於攻擊力 150% 的傷害。
  公式：與天賦 atk_scale、模組 atk_scale 相乘。
  預設：缺少或小於等於 0 時使用 1。

damage_scale
  意義：最終傷害倍率乘區。
  單位：倍率，1.2 = 最終傷害 *1.2。
  公式：技能、天賦、模組 damage_scale 彼此相乘，最後乘到傷害。
  預設：缺少或小於等於 0 時使用 1。

def
  意義：降低敵方防禦。
  值域：
  - -1 < 值 < 0：比例降低。例如 -0.2 = 防禦 -20%。
  - 值 <= -1：固定值降低。例如 -100 = 防禦 -100。
  - 0 或正數：目前公式不視為降低。
  公式：敵方防禦先套比例，再加固定負值，最低為 0。
  警告：資料源中的 def 也可能代表提升自身防禦，這種情況必須排除或 custom，不可直接套用。

def_penetrate_fixed
  意義：固定無視敵方防禦。
  單位：固定數值，通常使用正數。
  公式：敵方防禦在降低計算後，再減去此數值，最低為 0。

magic_resistance
  意義：降低敵方法術抗性。
  值域：
  - -1 < 值 < 0：比例降低。例如 -0.1 = 法抗 -10%。
  - 值 <= -1：固定點數降低。例如 -10 = 法抗 -10。
  公式：結果限制在 0 到 100。
  警告：同名 key 也常表示提升自身法抗，遇到該語意必須排除。

base_attack_time
  意義：攻擊間隔的秒數修正。
  單位：秒，可為負數。
  公式：(基礎攻擊間隔 + 技能修正 + 天賦修正) / 攻速倍率。
  例：-0.9 表示基礎攻擊間隔減少 0.9 秒。

attack_speed
  意義：攻擊速度修正。
  單位：攻速點數，20 表示 +20 攻速，不是 20%。
  公式：總攻速 = 面板攻速 + 技能 + 天賦 + 模組。

ATTACK_COUNT
  意義：每次攻擊同時造成的傷害段／命中數。
  單位：次數。
  預設：0 會正規化為 1。
  公式：每次攻擊傷害 * ATTACK_COUNT。
  注意：原始 cnt 不會全域自動映射，因為 cnt 可能表示目標數、堆疊數或觸發次數。

CHANGE_duration
  意義：覆蓋技能持續時間。
  單位：秒。
  公式：非 0 時取代 skillData.duration。

ADJUST_duration
  意義：在技能持續時間上加減秒數。
  單位：秒，可為負數。
  公式：有效技能時間 = max(1, 基準技能時間 + ADJUST_duration)。
  注意：基準技能時間會先套用 CHANGE_duration；若只需要扣除無輸出前搖，優先使用此欄位。

times
  意義：明確指定整個技能造成幾次攻擊／傷害段。
  單位：次數。
  公式：技能總傷 = 單次完整傷害 * times。
  DPS：因無可靠動畫時間，目前 DPS 直接等於總傷。

MAIN_times
  意義：只指定主傷害流的總命中次數，不影響額外傷害流。
  用途：技能同時包含一次直接傷害與一段持續額外傷害時，避免主傷害被持續時間重複計算。

attack@trigger_time
  意義：彈藥數量。
  單位：發數。
  公式：技能總傷 = 單次完整傷害 * 彈藥數量。
  DPS：總傷 / (實際攻擊間隔 * 彈藥數量)。

CHANGE_OTHER_attackType
  意義：額外獨立傷害流的傷害類型。
  常用值：物理、法術。

OTHER_atk_scale
  意義：額外獨立傷害流的攻擊倍率。
  單位：倍率。
  公式：沿用主傷害流的攻擊力加成、攻擊倍率與最終傷害倍率，再額外乘此倍率。
  特例：目前值 >= 10 時會被視為 custom 提供的固定攻擊力／固定傷害基準，而非倍率。

OTHER_base_attack_time
  意義：額外獨立傷害流自己的觸發間隔。
  單位：秒。
  缺少或為 0：沿用主攻擊間隔。

OTHER_times
  意義：只指定額外傷害流的總命中次數。
  用途：技能期間正常攻擊，但另有只觸發一次的額外傷害。

OTHER_duration
  意義：額外傷害流自己的有效持續時間。
  用途：主傷害只命中一次，但額外流會持續數秒的技能。

HEAL_heal_scale
  意義：治療技能的單次治療倍率。
  預設：缺少時視為 1，不改變以攻擊力為基準的治療量。
  用途：原始 heal_scale、attack@heal_scale 或巢狀治療倍率。

HEAL_flat
  意義：固定治療量，直接加到單次治療量。
  用途：資料明確給出固定治療值時。

HEAL_times
  意義：明確指定治療次數。
  用途：瞬發治療一次、五連發治療等不應依持續時間推算的技能。

HEAL_interval
  意義：治療流自己的觸發間隔。
  單位：秒。
  用途：每秒治療或其他 HoT。

HEAL_damage_scale
  語意：以已計算出的實際傷害量回饋治療。
  例子：0.5 = 治療量為造成傷害的 50%。
  常見於孑、夜魔等「造成傷害後按傷害量治療」的技能。
HEAL_max_hp_ratio
  語意：以幹員最終最大生命值計算固定週期恢復。
  例子：0.1 = 每次恢復最大生命值 10%。
  常見於宴、蛇屠箱等自我恢復技能。
HEAL_interval_add
  語意：治療流專用的攻擊間隔加算秒數。
  例子：1.3 = 以原攻擊間隔再加 1.3 秒。
  常見於技能文案沒有明寫，但實際開技後治療間隔變慢的停止攻擊治療技能。
HEAL_interval_scale
  語意：治療流專用的攻擊間隔倍率。
  例子：1.3 = 以原攻擊間隔的 130% 作為治療間隔。
  常見於守護者停止攻擊治療類技能。
HEAL_duration
  意義：治療流自己的有效持續時間。
  單位：秒。
  用途：持續治療時間不同於技能 duration 時。


二、常見但不可直接視為通用公式 key
------------------------------------

attack@atk_scale、attack@s2_atk_scale、attack@s2c.atk_scale
  常見語意：某個攻擊節點的攻擊倍率。
  建議：確認描述後 custom 映射為 atk_scale 或 OTHER_atk_scale。

cnt、attack@cnt
  可能語意：命中數、目標數、堆疊數、觸發次數。
  建議：只有確定代表每次攻擊段數時，才映射為 ATTACK_COUNT；若代表全技能次數則映射為 times。

duration、attack@duration
  可能語意：技能持續時間、效果持續時間、控制時間或單次 buff 時間。
  建議：只有確定代表整個傷害技能持續時間時，才映射為 CHANGE_duration；
  若是對原始技能時間做固定秒數增減，則映射為 ADJUST_duration。

interval、attack@interval
  可能語意：攻擊間隔、額外傷害間隔、觸發冷卻。
  建議：依語意映射為 base_attack_time 的修正，或 OTHER_base_attack_time 的絕對間隔。

prob、attack@prob、talent@prob
  意義通常是觸發機率。
  目前公式不會自動按機率取期望值。條件效果開啟時，也只有已寫入規則的效果才視為觸發。

heal_scale、attack@heal_scale、attack@atk_to_hp_recovery_ratio
  與治療量相關，目前技能傷害公式不計算治療量。

max_target、attack@max_target
  通常是可攻擊目標數，不等於單一目標被命中的次數，不應直接映射為 ATTACK_COUNT。

atk_scale_2、multi_atk_scale、multi_times
  語意依技能高度不一致。必須查描述與數值後再決定是主倍率、額外傷害流或總攻擊次數。


三、目前公式乘區順序摘要
------------------------
面板攻擊力
* (1 + 技能 atk + 特殊天賦 attack + 模組／分支 atk)
* 技能 atk_scale
* 天賦 atk_scale
* 模組／分支 atk_scale
* 額外傷害流 OTHER_atk_scale（只有額外流）
= 攻擊力基準

之後：
- 物理傷害：攻擊力基準 - 計算後敵方防禦。
- 法術傷害：攻擊力基準 * (100 - 計算後敵方法抗) / 100。
- 套用最低傷害比例 ensure_damage。
- 最後乘技能、天賦、模組 damage_scale。
