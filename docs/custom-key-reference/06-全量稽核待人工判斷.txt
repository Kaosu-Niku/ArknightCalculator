全量 JSON 稽核：待人工判斷案例
===============================

更新日期：2026-06-27

說明
----
本清單只保留無法由 JSON 文案安全決定單一目標傷害或治療的案例；已能確定的停攻、
多段、巢狀倍率、機率期望值、錯誤通用 key 與明確治療技能已直接加入 custom。
已完成案例統一記錄於 05-Custom適配案例總覽.txt。

技能時序與目標假設
------------------
- 澄闪「澄净闪耀」：已確認每個浮游單元各自獨立累積增傷層數；目前公式只有
  共用傷害流，待加入浮游單元各自暖機與自爆期望值後再處理。
- 缄默德克萨斯「阵雨连绵」：部署瞬間的一次傷害與後續二連擊段數不同；現行
  ATTACK_COUNT 會作用於所有傷害流，需先增加每條傷害流獨立段數。
- 琳琅诗怀雅「千金一掷」：持續輸出可算二連擊，但主動關閉時的金幣攻擊取決於
  人工關閉時間與實際金幣數；目前只計持續輸出。
- 丰川祥子「满月的舞会」：Fever 的二連擊需確認要按全技能覆蓋、指定覆蓋率，
  或只在條件開關開啟時視為全程生效；目前所有 Fever 內容均保留。

依敵方或外部行為觸發
--------------------
- 菲莱「冥河诅咒」、年「铜印」、棘刺「护身尖刺」屬受擊反擊，目前總傷為 0。
- 三角初华「我思念的」：目標受傷害時治療固定值，觸發依賴敵人或外部傷害行為；
  依瑪恩納反傷同策略，暫不加入治療表計算。
- 召喚物、替身、自動裝置、投擲物或道具裝置獨立行動造成的傷害與治療，仍依既定
  政策不併入幹員技能總傷或總治療。

治療技能候選
------------
以下是治療文案掃描後仍不適合直接套入第一版 HPS 公式的案例。

特殊或需決策：
- 森西「团体魔物大餐」：守護者，先持續恢復再一次治療；與缄默德克萨斯類似，
  需要多治療流或分段治療能力，先保留。
- 三角初华「我悲悯的」：吟遊者，週期造成傷害並治療友方；需同時處理傷害流與治療流。
- 锡人「“大拉里”」：鍊金師，投擲物區域內敵人受傷、友方恢復；依召喚物／投擲物政策先保留。
- 琳琅诗怀雅「仗义疏财」：行商，消耗金幣使下次攻擊治療；因金幣消耗與觸發次數過於特殊，先保留。
- 引星棘刺「度算浪波」：鍊金師，投擲友方區域並每秒回復；依召喚物／投擲物政策先保留。

尚需公式能力
------------
- 天賦固定 DOT：蓝毒、深巡等，需要天賦可建立獨立傷害流。
- 以敵方防禦為基準的附傷：刻俄柏「剥壳」，不能套用一般自身攻击力倍率。
- 天賦固定 DOT：棘刺、隐德来希等，同樣等待天賦獨立傷害流。
- 每條傷害流獨立段數與延遲起始時間。
- 元素損傷／元素傷害目前不與物理、法術、真實傷害混算。

稽核指令
--------
npm run custom:audit -- --summary

完整輸出會列出尚未覆蓋案例、命中的風險旗標、黑板 key 與技能／天賦文案。
掃描器採高召回設計，候選數量不代表錯誤數量；隊伍增益與純防禦效果也可能被列出。
